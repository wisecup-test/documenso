<rule_activation id="f05f7732-f8ec-4143-9b47-dd5ebe561352" title="Adopt Monorepo Package Structure with Scoped Modules: Packages Not Create" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files in the monorepo to enforce scoped package structure and prevent architectural violations.
</rule_activation>

### Rules

- **R-MONO-001** MUST_NOT: Packages MUST NOT create circular dependencies between each other.
- **R-MONO-002** MUST: All TypeScript/JavaScript modules intended for reuse across multiple application components MUST be organized under the packages/ directory structure.
- **R-MONO-003** MUST: API contract definitions, OpenAPI specifications, and versioned API interfaces MUST be located in packages/api with versioned subdirectories.
- **R-MONO-004** MUST: Server-only utilities including authentication, authorization, and data access logic MUST be isolated in packages/lib/server-only.
- **R-MONO-005** MUST_NOT: Client-side code MUST NOT import from packages/lib/server-only.
- **R-MONO-006** SHOULD: Use TypeScript path mapping in tsconfig.json to enable clean imports (e.g., '@packages/api/v1/contract' instead of relative paths).
- **R-MONO-007** SHOULD: Establish naming conventions for packages: packages/api for public contracts, packages/lib for shared utilities, packages/app-* for application-specific code.
- **R-MONO-008** SHOULD: Document the purpose and scope of each package in its own README.md file to guide developers.
- **R-MONO-009** MAY: Use tools like Turborepo, Nx, or Lerna to manage monorepo builds, caching, and dependency orchestration.

### Verify

```bash
# Count TypeScript files in established package structure
find packages/ -type f -name '*.ts' | grep -E 'packages/(api|lib|app-tests)' | wc -l

# Verify no client-side imports of server-only code
grep -r 'packages/lib/server-only' --include='*.ts' --exclude-dir=node_modules | grep -v 'server-only' || echo 'No client-side imports of server-only code'

# Verify API contract structure exists
test -d packages/api/v1 && test -f packages/api/v1/openapi.ts && test -f packages/api/v1/contract.ts

# Validate dependency graph for circular dependencies (requires madge or dependency-cruiser)
madge --circular packages/ || echo 'Dependency graph validation required'
```

**Accept when:**
- All TypeScript files under packages/ directory follow the established structure with clear package boundaries (api, lib, app-tests)
- No imports of packages/lib/server-only code exist in client-side bundles or components
- API contracts and OpenAPI specifications are located in packages/api with versioned subdirectories
- Automated dependency graph validation passes without circular dependency errors
- Build-time checks confirm server-only code is not included in client bundles
- No circular dependencies are detected between packages
