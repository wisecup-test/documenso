<rule_activation id="6b65c2ef-4250-4843-ae14-314770daea03" title="Adopt Server-Only Module Pattern for Backend-Specific Library Code: Server Only Modules" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files in the codebase to enforce server-only module isolation and prevent accidental exposure of backend-specific code in client-side environments.
</rule_activation>

### Rules

- **R-SERVERONLY-001** MUST_NOT: Server-only modules MUST NOT be imported directly by client-side components or pages.
- **R-SERVERONLY-002** MUST: All database access layers and ORM queries MUST be placed in server-only modules.
- **R-SERVERONLY-003** MUST: Authentication and authorization logic including token validation MUST be placed in server-only modules.
- **R-SERVERONLY-004** MUST: PDF generation and document processing utilities MUST be placed in server-only modules.
- **R-SERVERONLY-005** MUST: File system operations and server-side I/O MUST be placed in server-only modules.
- **R-SERVERONLY-006** MUST: API route handlers that access server-only resources MUST be placed in server-only modules.
- **R-SERVERONLY-007** MUST: Server-side configuration and environment variable access MUST be placed in server-only modules.
- **R-SERVERONLY-008** MAY: Type-only imports from server-only modules are permitted for client-side type checking (EXC-001).

### Verify

```bash
# Count server-only imports within server-only directory
grep -r "from.*server-only" packages/lib/server-only/ | wc -l

# Find server-only files missing 'server-only' package import
find packages/lib/server-only -name '*.ts' -exec grep -L "'server-only'" {} \;

# Verify client build contains no server-only references
npm run build:client && ! grep -r "server-only" dist/client/
```

**Accept when:**
- All files in the server-only directory import the 'server-only' package at the top of the file
- Client-side build output contains no references to server-only module paths or code
- Attempting to import a server-only module in a client component results in a clear error message during development or build
- ESLint rules successfully detect and block server-only imports in client code
- CI/CD pipeline build verification analyzes client bundle contents and finds no server-only code

<enforcement>
Claude Code MUST NOT skip or defer verification of server-only module isolation. All rules in this activation are mandatory and must be checked before accepting changes that involve server-side code, authentication, database access, or client-side imports.
</enforcement>
