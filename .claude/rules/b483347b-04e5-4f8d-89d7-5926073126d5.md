<rule_activation id="b483347b-04e5-4f8d-89d7-5926073126d5" title="Adopt Server-Only Module Pattern for Backend-Specific Library Code: Pdf Generation File" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files in the codebase to enforce server-only module isolation for backend-specific functionality.
</rule_activation>

### Rules

- **R-SERVERONLY-001** SHOULD: PDF generation, file processing, and document manipulation logic SHOULD be isolated in server-only modules.
- **R-SERVERONLY-002** MUST: All database access layers and ORM queries MUST be placed in server-only modules.
- **R-SERVERONLY-003** MUST: Authentication and authorization logic including token validation MUST be isolated in server-only modules.
- **R-SERVERONLY-004** MUST: File system operations and server-side I/O MUST be placed in server-only modules.
- **R-SERVERONLY-005** MUST: API route handlers that access server-only resources MUST be isolated in server-only modules.
- **R-SERVERONLY-006** MUST: Server-side configuration and environment variable access MUST be placed in server-only modules.
- **R-SERVERONLY-007** MAY: Type-only imports from server-only modules MAY be used in client-side code for type checking (EXC-001).
- **R-SERVERONLY-008** MUST: All server-only module files MUST import the 'server-only' package at the top of the file to enable runtime enforcement.
- **R-SERVERONLY-009** SHOULD: Server-only code SHOULD be organized under a consistent path structure like 'packages/lib/server-only/{domain}/' where domain represents functional areas.
- **R-SERVERONLY-010** SHOULD: TypeScript path aliases SHOULD be configured to make server-only imports explicit and easily identifiable in code reviews.

### Verify

```bash
# Count server-only imports in server-only directory
grep -r "from.*server-only" packages/lib/server-only/ | wc -l

# Find server-only files missing 'server-only' package import
find packages/lib/server-only -name '*.ts' -exec grep -L "'server-only'" {} \;

# Verify client build contains no server-only references
npm run build:client && ! grep -r "server-only" dist/client/
```

**Accept when:**
- All files in the server-only directory import the 'server-only' package at the top of the file
- Client-side build output contains no references to server-only module paths or code
- Attempting to import a server-only module in a client component results in a clear error message during development or build
- ESLint rules successfully detect and prevent server-only imports in client code
- CI/CD pipeline build verification analyzes client bundle contents and finds no server-only code

<enforcement>
Claude Code MUST NOT skip or defer verification of server-only module isolation. All R-SERVERONLY rules MUST be checked before approving changes to backend-specific library code, PDF generation files, authentication logic, or database access layers.
</enforcement>
