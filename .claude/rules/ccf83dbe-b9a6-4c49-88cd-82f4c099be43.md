<rule_activation id="ccf83dbe-b9a6-4c49-88cd-82f4c099be43" title="Adopt Prisma ORM as Primary Datastore Access Layer: Database Access Trpc" applies_to="**/*">
These rules are ALWAYS ACTIVE for all database access patterns in tRPC server routers and data access layers. All new database queries MUST follow the patterns established herein.
</rule_activation>

### Rules

- **R-PRISMA-001** MUST: All database access in tRPC routers MUST use Prisma Client as the primary ORM interface.
- **R-PRISMA-002** MUST: All tRPC router implementations requiring database access use Prisma Client.
- **R-PRISMA-003** MUST: Service layer functions that perform CRUD operations use Prisma Client.
- **R-PRISMA-004** MUST: Background jobs and scheduled tasks that interact with the primary datastore use Prisma Client.
- **R-PRISMA-005** MUST: Data migration scripts and seed files use Prisma Client.
- **R-PRISMA-006** MUST: API endpoints that query or mutate application data use Prisma Client.
- **R-PRISMA-007** MAY: Raw SQL usage is permitted only under documented exceptions (EXC-001: database-specific features not yet supported by Prisma; EXC-002: performance profiling demonstrates 3x+ slowdown in critical path operations).
- **R-PRISMA-008** MUST: Raw SQL usage requires inline code comments explaining the exception and corresponding tracking issues.
- **R-PRISMA-009** SHOULD: Maintain a prisma/schema.prisma file at project root defining all database models, relations, and indexes.
- **R-PRISMA-010** SHOULD: Configure Prisma Client generation in the build pipeline to run after schema changes and before TypeScript compilation.
- **R-PRISMA-011** SHOULD: Create a shared database context or service that instantiates and exports a singleton Prisma Client instance for use across all routers.

### Verify

```bash
# Verify Prisma Client imports in tRPC server routers
grep -r 'from @prisma/client' packages/trpc/server --include='*.ts' | wc -l

# Verify schema file exists
test -f prisma/schema.prisma && echo 'Schema file exists' || echo 'Schema file missing'

# Verify raw SQL usage is documented
grep -r 'executeRaw\|queryRaw' packages/trpc/server --include='*.ts' | wc -l
```

**Accept when:**
- All tRPC router files in the detected pattern (find-attachments, webhook-router, get-envelope-field) import and use PrismaClient
- A prisma/schema.prisma file exists at the project root defining all database models
- Raw SQL usage (if any) is documented with inline comments explaining the exception and has corresponding tracking issues
- No direct database driver imports (e.g., 'pg', 'mysql2') are detected outside of Prisma configuration
