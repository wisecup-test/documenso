<rule_activation id="01b98dd3-3b32-411f-9d6e-b9abc038d638" title="Adopt Remix File-Based Routing with URL Parameter Conventions for Authenticated Routes: Routes Use Nested" applies_to="**/*">
These rules are ALWAYS ACTIVE for all authenticated web application routes in the Remix application, organization-scoped resource access patterns, team-scoped resource access patterns, administrative interface routes, and settings, members, teams, groups, templates, folders, and document management routes.
</rule_activation>

### Rules

- **R-REMIX-001** MAY: Routes MAY use nested folder structures with + suffix to create layout boundaries that share common UI components.

### Verify

```bash
# Count authenticated routes with org and team URL patterns
find apps/remix/app/routes/_authenticated+ -name '*.tsx' | grep -E 'o\.\$orgUrl|t\.\$teamUrl' | wc -l

# Verify route loaders extract and validate parameters
grep -r 'export.*loader' apps/remix/app/routes/_authenticated+ | grep -c 'params'

# Count index routes using proper naming convention
find apps/remix/app/routes/_authenticated+ -name '*_index.tsx' | wc -l
```

**Accept when:**
- All authenticated routes are located within the _authenticated+ directory segment
- Organization routes use o.$orgUrl pattern and team routes use t.$teamUrl+ pattern consistently
- Index routes use _index.tsx suffix and layout boundaries use + suffix appropriately
- Route loaders properly extract and validate URL parameters before accessing resources

<enforcement>
Claude Code MUST NOT skip or defer verification. All routes must comply with the nested folder structure convention with + suffix for layout boundaries and consistent URL parameter naming patterns ($orgUrl, $teamUrl). Violations must be flagged during code review and CI pipeline checks must enforce compliance.
</enforcement>
