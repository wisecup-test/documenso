<rule_activation id="efac9f05-66e4-4589-948e-8cedabd8ee08" title="Adopt Prisma ORM as Primary Datastore Access Layer: Schema Changes Managed" applies_to="**/*">
These rules are ALWAYS ACTIVE for all database access patterns in tRPC server routers and data access layers. All new database queries MUST follow the patterns established herein.
</rule_activation>

### Rules

- **R-PRISMA-001** MUST: Schema changes MUST be managed through Prisma migrations rather than manual database alterations.
- **R-PRISMA-002** MUST: All tRPC server router implementations (envelope-router, webhook-router, etc.) MUST use Prisma Client for database access.
- **R-PRISMA-003** MUST: Database queries for attachments, envelope fields, webhooks, and related entities MUST use Prisma's query builder methods (findMany, findUnique, create, update, delete).
- **R-PRISMA-004** MUST: CRUD operations on primary application data models MUST be performed through Prisma Client.
- **R-PRISMA-005** SHOULD: Use Prisma's 'include' and 'select' options strategically to fetch only required relations and fields, optimizing query performance.
- **R-PRISMA-006** SHOULD: Leverage Prisma's middleware feature for cross-cutting concerns like soft deletes, audit logging, or tenant isolation.
- **R-PRISMA-007** SHOULD: Initialize Prisma Client as a singleton in the tRPC context to avoid connection pool exhaustion; ensure proper cleanup in serverless environments.
- **R-PRISMA-008** MAY: Raw SQL queries are permitted only when database-specific features (e.g., PostgreSQL full-text search, advanced JSON operations) are not supported by Prisma's query builder (EXC-002).
- **R-PRISMA-009** MAY: Raw SQL queries are permitted only when performance profiling demonstrates that Prisma query builder introduces unacceptable latency (>100ms overhead) for specific high-traffic endpoints (EXC-001).

### Verify

```bash
# Verify Prisma Client imports in router files
grep -r 'from @prisma/client' packages/trpc/server --include='*.ts' | wc -l

# Verify Prisma method calls in database operations
grep -r 'prisma\.' packages/trpc/server --include='*.ts' | grep -v 'node_modules' | wc -l

# Verify no direct database driver imports outside Prisma configuration
find packages/trpc/server -name '*.ts' -exec grep -l 'pg\|mysql\|sqlite' {} \; | grep -v prisma | wc -l
```

**Accept when:**
- All tRPC router files import PrismaClient and use it for database operations (verify command 1 shows imports in router files)
- Database queries use Prisma's query builder methods (findMany, findUnique, create, update, delete) rather than raw SQL (verify command 2 shows prisma method calls)
- No direct database driver imports (pg, mysql2, sqlite3) exist outside of Prisma configuration files (verify command 3 returns 0 or only shows Prisma-related files)
