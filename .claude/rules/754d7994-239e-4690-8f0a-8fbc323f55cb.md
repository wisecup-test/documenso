<rule_activation id="754d7994-239e-4690-8f0a-8fbc323f55cb" title="Adopt Remix File-Based Routing with URL Parameter Conventions for Authenticated Routes: Authenticated Routes Placed" applies_to="apps/remix/app/routes/**/*">
These rules are ALWAYS ACTIVE for all authenticated routes in the Remix application. All authenticated routes MUST be placed within the _authenticated+ route segment to enforce authentication boundaries.
</rule_activation>

### Rules

- **R-AUTH-001** MUST: All authenticated routes MUST be placed within the _authenticated+ route segment to enforce authentication boundaries.
- **R-AUTH-002** MUST: Organization-scoped routes MUST use the o.$orgUrl pattern consistently.
- **R-AUTH-003** MUST: Team-scoped routes MUST use the t.$teamUrl+ pattern consistently.
- **R-AUTH-004** MUST: Index routes MUST use _index.tsx suffix.
- **R-AUTH-005** MUST: Layout boundaries MUST use + suffix appropriately.
- **R-AUTH-006** MUST: Route loaders MUST properly extract and validate URL parameters before accessing resources.
- **R-AUTH-007** SHOULD: Use Remix's useParams() hook to extract $orgUrl and $teamUrl parameters in loaders and components.
- **R-AUTH-008** SHOULD: Implement shared layout components at the _authenticated+, o.$orgUrl, and t.$teamUrl+ boundaries.
- **R-AUTH-009** SHOULD: Create utility functions for generating type-safe route URLs to avoid hardcoding paths.
- **R-AUTH-010** SHOULD: Document the route structure in a central location (e.g., ROUTES.md) with examples.

### Verify

```bash
# Count authenticated routes with org and team patterns
find apps/remix/app/routes/_authenticated+ -name '*.tsx' | grep -E 'o\.\$orgUrl|t\.\$teamUrl' | wc -l

# Verify loaders extract params
grep -r 'export.*loader' apps/remix/app/routes/_authenticated+ | grep -c 'params'

# Count index routes
find apps/remix/app/routes/_authenticated+ -name '*_index.tsx' | wc -l

# Verify no authenticated routes exist outside _authenticated+
find apps/remix/app/routes -name '*.tsx' -not -path '*_authenticated+*' | grep -E 'o\.\$orgUrl|t\.\$teamUrl' | wc -l
```

**Accept when:**
- All authenticated routes are located within the _authenticated+ directory segment
- Organization routes use o.$orgUrl pattern and team routes use t.$teamUrl+ pattern consistently
- Index routes use _index.tsx suffix and layout boundaries use + suffix appropriately
- Route loaders properly extract and validate URL parameters before accessing resources
- No authenticated routes accessing protected resources exist outside the _authenticated+ boundary

Claude Code MUST NOT skip or defer verification. All routes must comply with R-AUTH-001 through R-AUTH-006 (MUST-level rules) before acceptance. Violations in CI pipeline must fail the build.
