<rule_activation id="ad15baf6-e767-413b-9ebd-e42c352728b2" title="Establish Type-Safe Public API Contracts for Library Modules: Public Endpoints Define" applies_to="packages/**/*.ts">
These rules are ALWAYS ACTIVE for all public API endpoints and library modules within the codebase. This ADR establishes type-safe public API contracts for library modules to prevent runtime errors and ensure type safety across package boundaries.
</rule_activation>

### Rules

- **R-API-001** MUST: All public API endpoints MUST define explicit TypeScript contracts using interface or type definitions.

### Verify

```bash
# Count contract definitions across packages
grep -r "export.*contract" packages/*/contract.ts | wc -l

# Find and verify OpenAPI and contract files exist
find packages/api -name 'openapi.ts' -o -name 'contract.ts' | xargs grep -l 'export'

# Verify TypeScript strict mode compilation passes
tsc --noEmit --strict && echo 'Type checking passed'

# Verify server-only modules are properly isolated
grep -r "server-only" packages/lib | grep -v node_modules
```

**Accept when:**
- All public API endpoints have corresponding TypeScript contract definitions with complete type coverage
- TypeScript strict mode compilation passes without type errors in contract files
- OpenAPI specifications exist for versioned API endpoints (v1, v2, etc.)
- Server-only modules are properly isolated and cannot be imported by client-side code

<enforcement>
Claude Code MUST NOT skip or defer verification. TypeScript compiler in strict mode during CI builds, ESLint rules, code review checklists, and automated tests are mandatory enforcement mechanisms. Pull requests MUST be blocked if new public APIs lack contract definitions.
</enforcement>
