### Rules

- **R-API-001** MUST: All API responses MUST support internationalization through Lingui configuration with locale-aware content.
- **R-API-002** MUST: All authenticated routes MUST use _authenticated+ prefix and implement proper authorization checks.
- **R-API-003** MUST: At least 90% of API routes MUST follow established URL parameter conventions ($orgUrl, $teamUrl).
- **R-API-004** MUST: Lingui configuration MUST be present and integrated into route handlers for internationalized responses.
- **R-API-005** MUST: PDF generation utilities MUST be isolated in server-only packages with appropriate content-type handling.
- **R-API-006** MUST: Use Remix's loader functions for GET requests and action functions for mutations, ensuring proper HTTP method semantics.
- **R-API-007** MUST: Implement shared authentication logic in _layout.tsx files to avoid duplication across route handlers.
- **R-API-008** MUST: Configure Lingui with appropriate locale detection strategy (Accept-Language header, user preferences, URL parameters).
- **R-API-009** SHOULD: For PDF generation, use streaming responses where possible to reduce memory footprint and improve time-to-first-byte.
- **R-API-010** SHOULD: Document URL parameter conventions in API specification (OpenAPI/Swagger) for external consumers.
- **R-API-011** SHOULD: Implement rate limiting and request validation middleware at the route level for public endpoints.

### Verify

```bash
# Count authenticated routes with proper prefix
grep -r '_authenticated+' apps/remix/app/routes/ | wc -l

# Find routes implementing loader or action functions
find apps/remix/app/routes -name '*.tsx' -exec grep -l 'loader\|action' {} \; | wc -l

# Verify Lingui configuration presence
grep -r 'lingui' . --include='*.config.*' --include='*.tsx' | head -5

# Locate PDF generation utilities in server-only packages
find packages/lib/server-only -name '*pdf*.ts' -type f
```

- PDF generation utilities are isolated in server-only packages with appropriate content-type handling
- TypeScript compilation succeeds without errors on all loader and action function signatures
- Integration tests validate authentication boundaries and authorization logic
