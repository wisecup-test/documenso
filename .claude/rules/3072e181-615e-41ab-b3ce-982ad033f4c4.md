<rule_activation id="3072e181-615e-41ab-b3ce-982ad033f4c4" title="Enforce Server-Only Code Isolation for Runtime Environment Boundaries: Server Only Modules" applies_to="**/*">
These rules are ALWAYS ACTIVE for all server-side code modules and apply to runtime environment boundary enforcement across the codebase.
</rule_activation>

### Rules

- **R-SOI-001** MUST: Server-only modules MUST include runtime guards or build-time checks that prevent execution or bundling in client environments.

### Verify

```bash
# Verify no client-side code imports from server-only directories
grep -r "from.*server-only" apps/*/client apps/*/components --include="*.ts" --include="*.tsx" || echo 'No client imports of server-only code found'

# Verify all server-only files have guards
find packages/lib/server-only -type f -name '*.ts' -exec grep -L "'server-only'" {} \; | wc -l | grep -q '^0$' && echo 'All server-only files have guards'

# Verify build does not bundle server code in client bundles
npm run build 2>&1 | grep -i 'server.*client.*bundle' && exit 1 || echo 'Build verification passed'
```

**Accept when:**
- No client-side code directly imports from server-only directories or modules
- All server-only modules include appropriate runtime guards or build-time enforcement markers (e.g., 'server-only' package or 'use server' directives)
- Build process successfully detects and rejects attempts to bundle server-only code in client bundles
- Code review checklist includes verification of server-only boundary compliance for new modules
- ESLint rules configured to detect restricted imports pass in CI pipeline
- Bundle analysis confirms no unexpected server dependencies in client builds

<enforcement>
Claude Code MUST NOT skip or defer verification. All server-only modules require explicit runtime guards or build-time checks. Violations block merge to main branch. Security team must be notified for violations involving credentials or secrets.
</enforcement>
