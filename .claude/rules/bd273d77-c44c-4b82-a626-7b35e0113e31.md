<rule_activation id="bd273d77-c44c-4b82-a626-7b35e0113e31" title="Standardize Public API Contract Design with Type-Safe Route Handlers and Internationalization: Document Generation Endpoints" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching the scope: Remix route handlers under apps/remix/app/routes/, server-side PDF generation utilities in packages/lib/server-only/, internationalization configuration files (lingui.config.ts), and authentication-gated endpoints for organizations, teams, and personal settings.
</rule_activation>

### Rules

- **R-API-001** MUST: Document generation endpoints (PDF, certificates, audit logs) MUST implement server-side rendering with appropriate Content-Type headers.
- **R-API-002** MUST: All authenticated routes use _authenticated+ prefix and implement proper authorization checks.
- **R-API-003** MUST: At least 90% of API routes follow the established URL parameter conventions ($orgUrl, $teamUrl).
- **R-API-004** MUST: Lingui configuration is present and integrated into route handlers for internationalized responses.
- **R-API-005** MUST: PDF generation utilities are isolated in server-only packages with appropriate content-type handling.
- **R-API-006** SHOULD: Use Remix's loader functions for GET requests and action functions for mutations, ensuring proper HTTP method semantics.
- **R-API-007** SHOULD: Implement shared authentication logic in _layout.tsx files to avoid duplication across route handlers.
- **R-API-008** SHOULD: Configure Lingui with appropriate locale detection strategy (Accept-Language header, user preferences, URL parameters).
- **R-API-009** SHOULD: For PDF generation, use streaming responses where possible to reduce memory footprint and improve time-to-first-byte.
- **R-API-010** SHOULD: Document URL parameter conventions in API specification (OpenAPI/Swagger) for external consumers.
- **R-API-011** SHOULD: Implement rate limiting and request validation middleware at the route level for public endpoints.

### Verify

```bash
# Count authenticated routes with _authenticated+ prefix
grep -r '_authenticated+' apps/remix/app/routes/ | wc -l

# Count routes implementing loader or action functions
find apps/remix/app/routes -name '*.tsx' -exec grep -l 'loader\|action' {} \; | wc -l

# Verify Lingui configuration presence and integration
grep -r 'lingui' . --include='*.config.*' --include='*.tsx' | head -5

# Locate PDF generation utilities in server-only packages
find packages/lib/server-only -name '*pdf*.ts' -type f
```

**Accept when:**
- All authenticated routes use _authenticated+ prefix and implement proper authorization checks
- At least 90% of API routes follow the established URL parameter conventions ($orgUrl, $teamUrl)
- Lingui configuration is present and integrated into route handlers for internationalized responses
- PDF generation utilities are isolated in server-only packages with appropriate content-type handling
- TypeScript compilation succeeds without errors on all route handlers
- Integration tests validate authentication boundaries and authorization logic
- All document generation endpoints implement appropriate Content-Type headers
