<rule_activation id="3f5685cc-2538-4b33-9144-0f1b8c341b3c" title="Adopt Remix File-Based Routing with URL Parameter Conventions for Authenticated Routes: Team Scoped Routes" applies_to="apps/remix/app/routes/**/*.tsx">
These rules are ALWAYS ACTIVE for all authenticated route files in the Remix application.
</rule_activation>

### Rules

- **R-REMIX-001** MUST: Team-scoped routes MUST use the $teamUrl dynamic parameter pattern within the t.$teamUrl+ segment (e.g., t.$teamUrl+/settings.members.tsx)
- **R-REMIX-002** MUST: All authenticated routes MUST be located within the _authenticated+ directory segment
- **R-REMIX-003** MUST: Organization routes MUST use o.$orgUrl pattern and team routes MUST use t.$teamUrl+ pattern consistently
- **R-REMIX-004** MUST: Index routes MUST use _index.tsx suffix and layout boundaries MUST use + suffix appropriately
- **R-REMIX-005** MUST: Route loaders MUST properly extract and validate URL parameters before accessing resources
- **R-REMIX-006** SHOULD: Use Remix's useParams() hook to extract $orgUrl and $teamUrl parameters in loaders and components
- **R-REMIX-007** SHOULD: Implement shared layout components at the _authenticated+, o.$orgUrl, and t.$teamUrl+ boundaries
- **R-REMIX-008** SHOULD: Create utility functions for generating type-safe route URLs to avoid hardcoding paths
- **R-REMIX-009** SHOULD: Document the route structure in a central location (e.g., ROUTES.md) with examples
- **R-REMIX-010** MAY: Consider using a route manifest generator to automatically create TypeScript types for all routes

### Verify

```bash
# Count team-scoped routes using $teamUrl pattern
find apps/remix/app/routes/_authenticated+ -name '*.tsx' | grep -E 't\.\\$teamUrl' | wc -l

# Verify loaders extract params
grep -r 'export.*loader' apps/remix/app/routes/_authenticated+ | grep -c 'params'

# Count index routes with proper suffix
find apps/remix/app/routes/_authenticated+ -name '*_index.tsx' | wc -l

# Verify all authenticated routes are in correct directory
find apps/remix/app/routes -name '*.tsx' -not -path '*_authenticated+*' | grep -v '^apps/remix/app/routes/\(_\|\[\|index\)' | wc -l
```

**Accept when:**
- All authenticated routes are located within the _authenticated+ directory segment
- Organization routes use o.$orgUrl pattern and team routes use t.$teamUrl+ pattern consistently
- Index routes use _index.tsx suffix and layout boundaries use + suffix appropriately
- Route loaders properly extract and validate URL parameters before accessing resources
- No protected resources are accessible from routes outside the _authenticated+ segment
