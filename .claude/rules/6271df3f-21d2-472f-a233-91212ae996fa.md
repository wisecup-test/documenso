<rule_activation id="6271df3f-21d2-472f-a233-91212ae996fa" title="Adopt Server-Only Data Access Layer with Transactional Boundaries: Data Access Functions" applies_to="**/*">
These rules are ALWAYS ACTIVE for all server-side data access operations. All data mutations and queries that interact with the database MUST follow the patterns defined herein.
</rule_activation>

### Rules

- **R-DAL-001** SHOULD: Data access functions SHOULD be organized by domain entity (e.g., folder/, recipient/, profile/, public-api/) to maintain clear module boundaries.
- **R-DAL-002** MUST: All database queries and mutations MUST be executed server-side only, never from client-side code.
- **R-DAL-003** MUST: All data mutation operations (create, update, delete) MUST be implemented as exported functions in the server-only layer with proper error handling.
- **R-DAL-004** MUST: Server-side API endpoints that interact with persistent storage MUST use the server-only data access layer.
- **R-DAL-005** MUST: Authentication and authorization checks MUST be enforced for all data access operations.
- **R-DAL-006** MUST: Data validation and transformation logic MUST occur before persistence.
- **R-DAL-007** SHOULD: Use TypeScript to define clear input/output types for each data access function.
- **R-DAL-008** SHOULD: Implement a consistent error handling strategy across all server-only functions using typed exceptions or result types.

### Verify

```bash
# Check for client-side database imports
grep -r 'import.*prisma\|import.*db' apps/*/app --exclude-dir=node_modules | grep -v 'server-only' | grep -v '.server.' || echo 'No client-side database imports detected'

# Verify minimum server-only modules exist
find packages/lib/server-only -type f -name '*.ts' | wc -l | awk '{if($1 >= 5) print "PASS: Found " $1 " server-only modules"; else print "FAIL: Expected at least 5 server-only modules"}'

# Verify data mutation functions are exported
grep -r 'export.*function.*delete\|export.*function.*create\|export.*function.*update' packages/lib/server-only --include='*.ts' | wc -l | awk '{if($1 >= 3) print "PASS: Found " $1 " data mutation functions"; else print "FAIL: Expected at least 3 mutation functions"}'
```

**Accept when:**
- No client-side code directly imports database clients or executes queries; all database access is mediated through server-only layer
- At least 5 server-only data access modules exist, organized by domain entity, each handling specific business operations
- All data mutation operations (create, update, delete) are implemented as exported functions in the server-only layer with proper error handling
- Database client imports (prisma, db) are only found in files within server-only directories or .server. files
