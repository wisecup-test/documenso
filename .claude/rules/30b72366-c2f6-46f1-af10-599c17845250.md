<rule_activation id="30b72366-c2f6-46f1-af10-599c17845250" title="Adopt Remix File-Based Routing with URL Parameter Conventions for Authenticated Routes: Organization Scoped Routes" applies_to="apps/remix/app/routes/_authenticated+/**/*.tsx">
These rules are ALWAYS ACTIVE for all authenticated route files in the Remix application using file-based routing with organization and team scoping.
</rule_activation>

### Rules

- **R-REMIX-001** MUST: Organization-scoped routes MUST use the $orgUrl dynamic parameter pattern (e.g., o.$orgUrl.settings.members.tsx)
- **R-REMIX-002** MUST: All authenticated routes MUST be located within the _authenticated+ directory segment
- **R-REMIX-003** MUST: Team-scoped routes MUST use the t.$teamUrl+ pattern consistently
- **R-REMIX-004** MUST: Index routes MUST use _index.tsx suffix and layout boundaries MUST use + suffix appropriately
- **R-REMIX-005** MUST: Route loaders MUST properly extract and validate URL parameters before accessing resources
- **R-REMIX-006** SHOULD: Use Remix's useParams() hook to extract $orgUrl and $teamUrl parameters in loaders and components
- **R-REMIX-007** SHOULD: Implement shared layout components at the _authenticated+, o.$orgUrl, and t.$teamUrl+ boundaries to provide consistent navigation and context
- **R-REMIX-008** SHOULD: Create utility functions for generating type-safe route URLs to avoid hardcoding paths
- **R-REMIX-009** SHOULD: Document the route structure in a central location (e.g., ROUTES.md) with examples of each pattern
- **R-REMIX-010** MAY: Consider using a route manifest generator to automatically create TypeScript types for all routes and their parameters

### Verify

```bash
# Count organization-scoped routes using $orgUrl pattern
find apps/remix/app/routes/_authenticated+ -name '*.tsx' | grep -E 'o\.\\$orgUrl|t\.\\$teamUrl' | wc -l

# Verify route loaders extract parameters
grep -r 'export.*loader' apps/remix/app/routes/_authenticated+ | grep -c 'params'

# Count index routes with proper naming
find apps/remix/app/routes/_authenticated+ -name '*_index.tsx' | wc -l

# Verify all authenticated routes are in _authenticated+ segment
find apps/remix/app/routes -name '*.tsx' -path '*_authenticated+*' | wc -l
```

**Accept when:**
- All authenticated routes are located within the _authenticated+ directory segment
- Organization routes use o.$orgUrl pattern and team routes use t.$teamUrl+ pattern consistently
- Index routes use _index.tsx suffix and layout boundaries use + suffix appropriately
- Route loaders properly extract and validate URL parameters before accessing resources
- No protected resources are accessible outside the _authenticated+ boundary
- URL parameter validation returns 404 for invalid combinations
