<rule_activation id="a288fe71-66ee-4de5-b226-7e18f31fbe4e" title="Standardize Logging Configuration for Environment-Aware Runtime Observability: Runtime Components Configure" applies_to="**/*">
These rules are ALWAYS ACTIVE for all runtime components that require logging and observability capabilities.
</rule_activation>

### Rules

- **R-LOG-001** MUST: All runtime components MUST configure logging behavior based on environment variables rather than hardcoded values.

### Verify

```bash
# Check for environment variable references in logging configuration
grep -r 'process\.env.*LOG' --include='*.ts' --include='*.tsx' --include='*.js' | wc -l

# Check for logger initialization and configuration patterns
grep -r 'logger\.configure\|createLogger\|winston\|pino' --include='*.ts' --include='*.tsx' | head -20

# Check for environment template files documenting logging configuration
find . -name '*.env.example' -o -name '*.env.template' | xargs grep -l 'LOG_LEVEL\|LOG_FORMAT' 2>/dev/null
```

**Accept when:**
- Environment variable references for logging configuration are found in multiple source files
- Logger initialization or configuration code is present in the codebase
- Environment template files document logging-related configuration variables

<enforcement>
Claude Code MUST NOT skip or defer verification of environment-based logging configuration. Code review checklist verification, automated linting rules, and CI pipeline tests are mandatory enforcement mechanisms.
</enforcement>
