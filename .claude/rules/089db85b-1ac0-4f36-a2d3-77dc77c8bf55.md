<rule_activation id="089db85b-1ac0-4f36-a2d3-77dc77c8bf55" title="Adopt Remix File-Based Routing with URL Parameter Conventions for Authenticated Routes: Index Routes Use" applies_to="apps/remix/app/routes/_authenticated+/**/*.tsx">
These rules are ALWAYS ACTIVE for all authenticated route files in the Remix application using file-based routing with URL parameter conventions.
</rule_activation>

### Rules

- **R-REMIX-001** MUST: Index routes MUST use the `_index.tsx` suffix to indicate default route behavior within a segment.
- **R-REMIX-002** MUST: All authenticated routes MUST be located within the `_authenticated+` directory segment.
- **R-REMIX-003** MUST: Organization routes MUST use the `o.$orgUrl` pattern consistently.
- **R-REMIX-004** MUST: Team routes MUST use the `t.$teamUrl+` pattern consistently.
- **R-REMIX-005** MUST: Layout boundaries MUST use the `+` suffix appropriately to indicate shared layout segments.
- **R-REMIX-006** MUST: Route loaders MUST properly extract and validate URL parameters before accessing resources.
- **R-REMIX-007** SHOULD: Use Remix's `useParams()` hook to extract `$orgUrl` and `$teamUrl` parameters in loaders and components.
- **R-REMIX-008** SHOULD: Implement shared layout components at the `_authenticated+`, `o.$orgUrl`, and `t.$teamUrl+` boundaries to provide consistent navigation and context.
- **R-REMIX-009** SHOULD: Create utility functions for generating type-safe route URLs to avoid hardcoding paths.
- **R-REMIX-010** MAY: Document the route structure in a central location (e.g., ROUTES.md) with examples of each pattern and when to use them.

### Verify

```bash
# Count authenticated routes with org/team patterns
find apps/remix/app/routes/_authenticated+ -name '*.tsx' | grep -E 'o\.\\$orgUrl|t\.\\$teamUrl' | wc -l

# Verify loaders extract params
grep -r 'export.*loader' apps/remix/app/routes/_authenticated+ | grep -c 'params'

# Count index routes using _index.tsx suffix
find apps/remix/app/routes/_authenticated+ -name '*_index.tsx' | wc -l

# Verify no protected routes exist outside _authenticated+
find apps/remix/app/routes -name '*.tsx' ! -path '*_authenticated+*' | xargs grep -l 'requireAuth\|loader.*auth' | wc -l
```

**Accept when:**
- All authenticated routes are located within the `_authenticated+` directory segment
- Organization routes use `o.$orgUrl` pattern and team routes use `t.$teamUrl+` pattern consistently
- Index routes use `_index.tsx` suffix and layout boundaries use `+` suffix appropriately
- Route loaders properly extract and validate URL parameters before accessing resources
- No protected resources are accessible from routes outside the `_authenticated+` boundary
- URL parameter validation returns 404 for invalid org/team combinations
