<rule_activation id="b2307cf5-d663-4b32-9da7-0beb89a9da66" title="Adopt Server-Only Module Pattern for Backend-Specific Library Code: Functions That Perform" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files in the codebase to enforce server-only module isolation for backend-specific functionality.
</rule_activation>

### Rules

- **R-SERVERONLY-001** SHOULD: Functions that perform authentication, authorization, or token validation SHOULD be implemented as server-only modules.
- **R-SERVERONLY-002** MUST: All database access layers and ORM queries MUST be placed in server-only modules.
- **R-SERVERONLY-003** MUST: All authentication and authorization logic including token validation MUST be placed in server-only modules.
- **R-SERVERONLY-004** MUST: PDF generation and document processing utilities MUST be placed in server-only modules.
- **R-SERVERONLY-005** MUST: File system operations and server-side I/O MUST be placed in server-only modules.
- **R-SERVERONLY-006** MUST: API route handlers that access server-only resources MUST be placed in server-only modules.
- **R-SERVERONLY-007** MUST: Server-side configuration and environment variable access MUST be placed in server-only modules.
- **R-SERVERONLY-008** MAY: Type-only imports from server-only modules MAY be used in client-side code for type checking (EXC-001).
- **R-SERVERONLY-009** MUST: All server-only module files MUST import the 'server-only' package at the top of the file to enable runtime enforcement.
- **R-SERVERONLY-010** MUST: Server-only code MUST be organized under a consistent path structure like 'packages/lib/server-only/{domain}/' where domain represents functional areas.

### Verify

```bash
# Count server-only imports in server-only directory
grep -r "from.*server-only" packages/lib/server-only/ | wc -l

# Find server-only files missing 'server-only' package import
find packages/lib/server-only -name '*.ts' -exec grep -L "'server-only'" {} \;

# Verify client build contains no server-only references
npm run build:client && ! grep -r "server-only" dist/client/
```

**Accept when:**
- All files in the server-only directory import the 'server-only' package at the top of the file
- Client-side build output contains no references to server-only module paths or code
- Attempting to import a server-only module in a client component results in a clear error message during development or build
- No server-only code is detected in client bundles during CI/CD verification

<enforcement>
Claude Code MUST NOT skip or defer verification. All server-only module patterns MUST be verified before accepting changes that involve authentication, authorization, database access, PDF generation, file system operations, or API route handlers.
</enforcement>
