<rule_activation id="b1541a4b-1957-41d2-90d9-b1ec666f24c5" title="Establish Type-Safe Public API Contracts for Library Modules: Public Contracts Not" applies_to="**/*">
These rules are ALWAYS ACTIVE for all public API contract definitions and library module interfaces within the codebase, particularly in /packages/ directories, public API endpoints, and shared library modules.
</rule_activation>

### Rules

- **R-API-001** MUST NOT: Public API contracts MUST NOT expose internal implementation details or private types.
- **R-API-002** MUST: Create contract.ts files in each package's public API directory to centralize type definitions.
- **R-API-003** MUST: Use TypeScript's 'export type' and 'export interface' to explicitly mark public contracts.
- **R-API-004** SHOULD: Leverage OpenAPI decorators or schema generators to keep OpenAPI specs synchronized with TypeScript types.
- **R-API-005** SHOULD: Implement pre-commit hooks to validate that all public API functions have corresponding contract definitions.
- **R-API-006** SHOULD: Document migration path for existing APIs without contracts, prioritizing high-traffic endpoints first.
- **R-API-007** MUST: Enforce semantic versioning and maintain changelog for breaking changes to public contracts.
- **R-API-008** SHOULD: Establish contract review checklist and require JSDoc comments on all public interfaces.
- **R-API-009** MUST: Implement automated tests that validate implementation against contracts and use integration tests to verify API behavior.

### Verify

```bash
# Count contract definitions across packages
grep -r "export.*contract" packages/*/contract.ts | wc -l

# Find and verify OpenAPI and contract files
find packages/api -name 'openapi.ts' -o -name 'contract.ts' | xargs grep -l 'export'

# Run TypeScript strict mode type checking
tsc --noEmit --strict && echo 'Type checking passed'

# Verify server-only modules are properly isolated
grep -r "server-only" packages/lib | grep -v node_modules
```

**Accept when:**
- All public API endpoints have corresponding TypeScript contract definitions with complete type coverage
- TypeScript strict mode compilation passes without type errors in contract files
- OpenAPI specifications exist for versioned API endpoints (v1, v2, etc.)
- Server-only modules are properly isolated and cannot be imported by client-side code
- All public API functions have JSDoc comments documenting their contracts
- Pre-commit hooks validate contract definitions for new APIs

<enforcement>
Claude Code MUST verify TypeScript compilation in strict mode passes for all contract files. MUST NOT allow public APIs without corresponding contract definitions. MUST enforce this rule during code review and CI validation. Violations block pull requests and CI builds.
</enforcement>
