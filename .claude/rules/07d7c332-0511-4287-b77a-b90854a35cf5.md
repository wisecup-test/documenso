<rule_activation id="07d7c332-0511-4287-b77a-b90854a35cf5" title="Adopt Remix File-Based Routing with URL Parameter Conventions for Authenticated Routes: Admin Routes Prefixed" applies_to="apps/remix/app/routes/**/*">
These rules are ALWAYS ACTIVE for all authenticated route files in the Remix application.
</rule_activation>

### Rules

- **R-REMIX-001** SHOULD: Admin routes SHOULD be prefixed with `admin+` to clearly distinguish administrative functionality from user-facing routes.
- **R-REMIX-002** SHOULD: All authenticated routes SHOULD be located within the `_authenticated+` directory segment.
- **R-REMIX-003** SHOULD: Organization routes SHOULD use the `o.$orgUrl` pattern consistently.
- **R-REMIX-004** SHOULD: Team routes SHOULD use the `t.$teamUrl+` pattern consistently.
- **R-REMIX-005** SHOULD: Index routes SHOULD use the `_index.tsx` suffix.
- **R-REMIX-006** SHOULD: Layout boundaries SHOULD use the `+` suffix appropriately.
- **R-REMIX-007** MUST: Route loaders MUST properly extract and validate URL parameters before accessing resources.
- **R-REMIX-008** MUST: All routes accessing protected resources MUST be located within `_authenticated+` or equivalent authentication boundary.

### Verify

```bash
# Count authenticated routes with org/team patterns
find apps/remix/app/routes/_authenticated+ -name '*.tsx' | grep -E 'o\.\\$orgUrl|t\.\\$teamUrl' | wc -l

# Verify loaders extract params
grep -r 'export.*loader' apps/remix/app/routes/_authenticated+ | grep -c 'params'

# Count index routes with proper suffix
find apps/remix/app/routes/_authenticated+ -name '*_index.tsx' | wc -l

# Check for admin+ prefixed routes
find apps/remix/app/routes/_authenticated+ -name 'admin+*' -type d | wc -l
```

**Accept when:**
- All authenticated routes are located within the `_authenticated+` directory segment
- Organization routes use `o.$orgUrl` pattern and team routes use `t.$teamUrl+` pattern consistently
- Index routes use `_index.tsx` suffix and layout boundaries use `+` suffix appropriately
- Route loaders properly extract and validate URL parameters before accessing resources
- Admin routes are prefixed with `admin+` to distinguish administrative functionality
- No protected resources are accessible outside the `_authenticated+` boundary

<enforcement>
Claude Code MUST NOT skip or defer verification. All route files MUST comply with naming conventions and parameter validation requirements before acceptance.
</enforcement>
