<rule_activation id="3bfe858f-8dcb-4b8a-9310-f9a564102666" title="Adopt Server-Only Module Pattern for External API Integrations: Server Only Modules" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching `**/*`. Server-only modules handling external API integrations, PDF generation, and sensitive operations must never execute in client-side contexts.
</rule_activation>

### Rules

- **R-SERVERONLY-001** MUST NOT: Server-only modules MUST NOT be imported or referenced directly from client-side code or components that execute in browser contexts.

### Verify

```bash
# Check for client-side server-only imports in Remix routes
grep -r "from.*server-only" apps/remix/app/routes --include="*.tsx" --include="*.ts" | grep -v "loader\|action" || echo "No client-side server-only imports detected"

# Count server-only modules exporting sensitive credentials
find packages/lib/server-only -type f -name "*.ts" -exec grep -l "export.*API_KEY\|SECRET\|PASSWORD" {} \; | wc -l

# Verify client bundle size and check for server-only code leakage
npm run build && du -sh dist/client/**/*.js | awk '{if($1 ~ /M/ && $1+0 > 5) print "Warning: Large client bundle detected: " $0}'
```

**Accept when:**
- All server-only modules reside under designated server-only directory structures (e.g., `packages/lib/server-only/`) and are not imported directly in client-side code
- Build process successfully prevents bundling of server-only code in client JavaScript outputs
- CI pipeline includes automated checks that fail on detection of server-only imports in client contexts
- All external API integrations, PDF operations, and audit log rendering are implemented as server-only modules
- ESLint rules and TypeScript path restrictions enforce server-only import boundaries
- Bundle analysis confirms no server-only code appears in client chunks

<enforcement>
Claude Code MUST NOT skip or defer verification. All server-only module imports must be validated against R-SERVERONLY-001 before approval. Build-time enforcement and CI checks are mandatory.
</enforcement>
