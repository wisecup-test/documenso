<rule_activation id="1899de65-b7e4-4ffc-a0fc-b22d84616ab5" title="Adopt Server-Only Data Access Layer with Specialized Query Functions: Database Access Operations" applies_to="**/*">
These rules are ALWAYS ACTIVE for all server-side data access operations. All database queries MUST be encapsulated in dedicated server-only modules following the established pattern.
</rule_activation>

### Rules

- **R-DAL-001** MUST: All database access operations MUST be encapsulated in dedicated modules within a server-only directory structure that prevents client-side bundling.
- **R-DAL-002** MUST: All database read operations (SELECT queries) be located in server-only modules.
- **R-DAL-003** MUST: All database write operations (INSERT, UPDATE, DELETE) be located in server-only modules.
- **R-DAL-004** MUST: All data access requiring authentication or authorization be encapsulated in server-only modules.
- **R-DAL-005** MUST: Each data access function follow the verb-entity-qualifier naming pattern (get-*, find-*, create-*, update-*, delete-*, *-internal).
- **R-DAL-006** MUST: Each function export a single default or named function with clear TypeScript types for parameters and return values.
- **R-DAL-007** SHOULD: Include JSDoc comments documenting authorization requirements, expected errors, and usage examples.
- **R-DAL-008** SHOULD: Use a shared database client/connection pool imported by all data access functions to ensure consistent configuration.
- **R-DAL-009** SHOULD: Establish code review guidelines to identify opportunities for consolidation and create shared query utilities for common patterns.

### Verify

```bash
# Count server-only module usage
grep -r 'server-only' --include='*.ts' --include='*.tsx' | grep -v 'node_modules' | wc -l

# Find server-only files missing function exports
find . -path '*/server-only/*' -name '*.ts' | xargs grep -L 'export.*function' | wc -l

# Detect direct database imports in client-side code
grep -r 'import.*prisma\|import.*db' --include='*.tsx' --exclude-dir='server-only' --exclude-dir='api' | grep -v 'node_modules' | wc -l
```

**Accept when:**
- All database access operations are located in server-only directories with no direct database imports in client-side code
- Each data access function follows the verb-entity-qualifier naming pattern and exports a single focused operation
- Code review confirms no database credentials or connection strings are exposed outside the server-only layer
- Automated tests verify that client bundles do not include server-only modules
- TypeScript module resolution and bundler configuration prevent server-only imports in client code
- No database credentials or connection strings appear outside the server-only layer

<enforcement>
Claude Code MUST verify all R-DAL rules are satisfied before approving server-side data access implementations. Violations result in build failures and code review rejection. Exceptions require architecture review board approval and must be documented with expiration dates.
</enforcement>
