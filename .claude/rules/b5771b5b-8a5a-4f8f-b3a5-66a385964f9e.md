<rule_activation id="b5771b5b-8a5a-4f8f-b3a5-66a385964f9e" title="Adopt Server-Side Rendering with Loader Functions for Route-Level Data Fetching: Routes Implement Client" applies_to="app/routes/**/*.{ts,tsx}">
These rules are ALWAYS ACTIVE for all Remix route modules in the application. All route files MUST implement server-side data loading patterns as specified herein.
</rule_activation>

### Rules

- **R-SSR-001** MUST: All route modules that require initial data MUST implement exported loader functions for server-side data fetching.
- **R-SSR-002** MUST: Route components MUST NOT use useEffect with fetch or axios for initial data loading; use useLoaderData() instead.
- **R-SSR-003** MUST: Loader functions MUST implement error handling using throw new Response() for HTTP errors or throw redirect() for authentication redirects.
- **R-SSR-004** MUST: Routes requiring authentication MUST check session/token validity in the loader and redirect to login if unauthorized.
- **R-SSR-005** SHOULD: Leverage Remix's built-in caching headers in loader responses to enable CDN and browser caching where appropriate.
- **R-SSR-006** SHOULD: Use defer() for streaming server-side rendering when some data can be loaded asynchronously after initial render.
- **R-SSR-007** SHOULD: Structure loader functions to be testable independently from components, enabling unit tests for data fetching logic.
- **R-SSR-008** MAY: Routes MAY implement client-side data fetching for progressive enhancement or real-time updates after initial render.

### Verify

```bash
# Count exported loader functions in route files
grep -r "export.*loader" app/routes/ --include="*.tsx" --include="*.ts" | wc -l

# Check for client-side data fetching patterns in route components
grep -r "useEffect.*fetch\|useEffect.*axios" app/routes/ --include="*.tsx" | wc -l

# Verify TypeScript compilation passes
npm run type-check -- --noEmit
```

**Accept when:**
- All route files that require initial data have exported loader functions (grep count > 0 for routes with data needs)
- No route components use useEffect for initial data fetching (grep count = 0 for useEffect with fetch/axios in routes)
- TypeScript compilation passes without errors, confirming type safety between loaders and components

<enforcement>
Claude Code MUST NOT skip or defer verification. All route modules MUST be checked for loader function implementation and absence of client-side data fetching patterns before accepting changes.
</enforcement>
