<rule_activation id="530db378-73c8-4160-a025-cb3558300a8d" title="Adopt Server-Only Module Pattern for Backend-Specific Library Code: Server Only Modules" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files in the codebase to enforce server-only module isolation and prevent accidental exposure of backend-specific code in client-side environments.
</rule_activation>

### Rules

- **R-SOM-001** MUST: Server-only modules MUST use the 'server-only' package or equivalent mechanism to throw runtime errors if imported in client environments.

### In Scope

The following code patterns and modules MUST follow server-only isolation:
- All database access layers and ORM queries
- Authentication and authorization logic including token validation
- PDF generation and document processing utilities
- File system operations and server-side I/O
- API route handlers that access server-only resources
- Server-side configuration and environment variable access

### Out of Scope

The following code patterns are NOT subject to server-only requirements:
- Shared TypeScript types and interfaces used for type checking
- Universal utility functions that work in both client and server environments
- Client-side API client libraries that make HTTP requests
- React components and UI logic
- Client-side state management code

### Exceptions

- **EXC-001**: Type-only imports from server-only modules are needed for client-side type checking and are permitted.

### Verify

```bash
# Verify all server-only modules import the 'server-only' package
find packages/lib/server-only -name '*.ts' -exec grep -L "'server-only'" {} \;

# Count server-only imports to track adoption
grep -r "from.*server-only" packages/lib/server-only/ | wc -l

# Verify client bundle contains no server-only references
npm run build:client && ! grep -r "server-only" dist/client/
```

**Accept when:**
- All files in the server-only directory import the 'server-only' package at the top of the file
- Client-side build output contains no references to server-only module paths or code
- Attempting to import a server-only module in a client component results in a clear error message during development or build
- No server-only code or credentials are present in the client bundle

<enforcement>
Claude Code MUST NOT skip or defer verification. All server-only module patterns MUST be validated before accepting changes to backend-specific library code. Violations MUST trigger build failures and security reviews.
</enforcement>
