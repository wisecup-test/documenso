<rule_activation id="2468dbdb-b8d6-4d93-8e8f-27835af2dbbf" title="Standardize API E2E Testing with Spec-Based Test Organization: Test Files Further" applies_to="packages/app-tests/e2e/api/**/*.spec.ts">
These rules are ALWAYS ACTIVE for all API E2E test files in the packages/app-tests/e2e/api directory structure.
</rule_activation>

### Rules

- **R-API-E2E-001** MAY: Test files MAY be further organized into logical groupings within version directories if the number of endpoints grows significantly.
- **R-API-E2E-002** MUST: All API E2E test files follow the `*.spec.ts` naming convention.
- **R-API-E2E-003** MUST: Test files are organized in the appropriate API type subdirectory (trpc/, v1/, or v2/).
- **R-API-E2E-004** MUST: Each spec file contains valid test definitions (describe/test/it blocks) and can be executed independently.
- **R-API-E2E-005** MUST: Each spec file does not rely on execution order or shared state from other spec files.
- **R-API-E2E-006** SHOULD: Use shared test fixtures and utilities from a common location (e.g., packages/app-tests/e2e/fixtures/) to reduce duplication across spec files.
- **R-API-E2E-007** SHOULD: When adding a new API endpoint, create a corresponding spec file following the {feature-name}.spec.ts naming convention.
- **R-API-E2E-008** SHOULD: Use test tags or metadata to categorize tests by feature area, enabling selective test execution during development.
- **R-API-E2E-009** MUST: When deprecating an API version, maintain its test suite until the version is fully removed from production.
- **R-API-E2E-010** MUST: Experimental or internal-only APIs that are exempted from E2E test requirements must be documented in TESTING_EXCEPTIONS.md with justification and expiration date.

### Verify

```bash
# Verify test files exist and are organized by API type
find packages/app-tests/e2e/api -name '*.spec.ts' -type f | grep -E '(trpc|v1|v2)/' | wc -l

# Verify required API type subdirectories exist
test -d packages/app-tests/e2e/api/trpc && test -d packages/app-tests/e2e/api/v1 && test -d packages/app-tests/e2e/api/v2

# Verify all spec files contain valid test definitions
find packages/app-tests/e2e/api -name '*.spec.ts' | xargs grep -l 'describe\|test\|it' | wc -l
```

**Accept when:**
- The packages/app-tests/e2e/api directory exists with subdirectories for trpc/, v1/, and v2/
- All API E2E test files follow the *.spec.ts naming convention and are organized in the appropriate API type subdirectory
- Each spec file contains valid test definitions (describe/test/it blocks) and can be executed independently
- Test files do not depend on execution order or shared state from other spec files
- Shared test utilities and fixtures are located in packages/app-tests/e2e/fixtures/ or similar common location
