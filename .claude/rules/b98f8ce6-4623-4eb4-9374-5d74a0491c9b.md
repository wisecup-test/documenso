<rule_activation id="b98f8ce6-4623-4eb4-9374-5d74a0491c9b" title="Adopt Server-Side Rendering with Loader Functions for Route-Level Data Fetching: Loader Functions Handle" applies_to="app/routes/**/*.{ts,tsx}">
These rules are ALWAYS ACTIVE for all Remix route modules in the application. All route files MUST implement server-side data loading patterns as specified herein.
</rule_activation>

### Rules

- **R-SSR-001** SHOULD: Loader functions SHOULD handle authentication, authorization, and data validation on the server before returning data.
- **R-SSR-002** MUST: All route files that require initial data MUST have exported loader functions.
- **R-SSR-003** MUST: Route components MUST NOT use useEffect with fetch or axios for initial data fetching.
- **R-SSR-004** SHOULD: Use Remix's useLoaderData() hook in route components to access data returned from loader functions with full type safety.
- **R-SSR-005** SHOULD: Implement error handling in loaders using throw new Response() for HTTP errors or throw redirect() for authentication redirects.
- **R-SSR-006** SHOULD: Leverage Remix's built-in caching headers in loader responses to enable CDN and browser caching where appropriate.
- **R-SSR-007** SHOULD: For routes requiring authentication, check session/token validity in the loader and redirect to login if unauthorized.
- **R-SSR-008** MAY: Use defer() for streaming server-side rendering when some data can be loaded asynchronously after initial render.
- **R-SSR-009** SHOULD: Structure loader functions to be testable independently from components, enabling unit tests for data fetching logic.

### Verify

```bash
# Count exported loader functions in route files
grep -r "export.*loader" app/routes/ --include="*.tsx" --include="*.ts" | wc -l

# Detect client-side data fetching patterns in route components
grep -r "useEffect.*fetch\|useEffect.*axios" app/routes/ --include="*.tsx" | wc -l

# Verify TypeScript compilation passes
npm run type-check -- --noEmit
```

**Accept when:**
- All route files that require initial data have exported loader functions (grep count > 0 for routes with data needs)
- No route components use useEffect for initial data fetching (grep count = 0 for useEffect with fetch/axios in routes)
- TypeScript compilation passes without errors, confirming type safety between loaders and components
