<rule_activation id="e83e769a-4cd6-4a27-97ac-ba1474de1d9a" title="Adopt Monorepo Package Structure with Scoped Modules: Test Related Constants" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files in the monorepo to enforce consistent package structure and architectural boundaries.
</rule_activation>

### Rules

- **R-MONO-001** SHOULD: Test-related constants and utilities SHOULD be organized in packages/app-tests to maintain separation from production code.
- **R-MONO-002** SHOULD: All TypeScript/JavaScript modules intended for reuse across multiple application components SHOULD be organized in the packages/ directory structure.
- **R-MONO-003** SHOULD: API contract definitions, OpenAPI specifications, and versioned API interfaces SHOULD be located in packages/api with versioned subdirectories.
- **R-MONO-004** SHOULD: Server-only utilities including authentication, authorization, and data access logic SHOULD be organized in packages/lib/server-only.
- **R-MONO-005** MUST: Client-side code MUST NOT import from packages/lib/server-only to prevent security vulnerabilities and reduce bundle size.
- **R-MONO-006** SHOULD: Shared type definitions and interfaces used across packages SHOULD be organized in appropriate scoped packages.
- **R-MONO-007** MAY: Use TypeScript path mapping in tsconfig.json to enable clean imports (e.g., '@packages/api/v1/contract' instead of relative paths).
- **R-MONO-008** MAY: Consider using tools like Turborepo, Nx, or Lerna to manage monorepo builds, caching, and dependency orchestration.

### Verify

```bash
# Count TypeScript files in packages directory following established structure
find packages/ -type f -name '*.ts' | grep -E 'packages/(api|lib|app-tests)' | wc -l

# Verify no client-side imports of server-only code
grep -r 'packages/lib/server-only' --include='*.ts' --exclude-dir=node_modules | grep -v 'server-only' || echo 'No client-side imports of server-only code'

# Verify API contract structure exists
test -d packages/api/v1 && test -f packages/api/v1/openapi.ts && test -f packages/api/v1/contract.ts
```

**Accept when:**
- All TypeScript files under packages/ directory follow the established structure with clear package boundaries (api, lib, app-tests)
- No imports of packages/lib/server-only code exist in client-side bundles or components
- API contracts and OpenAPI specifications are located in packages/api with versioned subdirectories
- Automated dependency graph validation passes without circular dependency errors
- Test-related constants are located in packages/app-tests and not mixed with production code

<enforcement>
Claude Code MUST verify package structure compliance before approving code changes. Violations of R-MONO-005 (server-only imports in client code) MUST block acceptance. Dependency graph validation MUST run on every relevant change.
</enforcement>
