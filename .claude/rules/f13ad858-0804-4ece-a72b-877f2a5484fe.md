<rule_activation id="f13ad858-0804-4ece-a72b-877f2a5484fe" title="Standardize Public API Contract Design with Type-Safe Route Handlers and Internationalization: Settings Related Endpoints" applies_to="apps/remix/app/routes/**/*.tsx">
These rules are ALWAYS ACTIVE for all Remix route handlers implementing public-facing API endpoints with authentication, settings management, and internationalization support.
</rule_activation>

### Rules

- **R-API-001** SHOULD: Settings-related endpoints SHOULD follow the pattern `o.$orgUrl.settings.{domain}` for organizational settings and `settings+/_dynamic_personal_routes+` for personal settings.
- **R-API-002** MUST: All authenticated routes MUST use `_authenticated+` prefix and implement proper authorization checks.
- **R-API-003** MUST: Use Remix's loader functions for GET requests and action functions for mutations, ensuring proper HTTP method semantics.
- **R-API-004** SHOULD: Implement shared authentication logic in `_layout.tsx` files to avoid duplication across route handlers.
- **R-API-005** SHOULD: Configure Lingui with appropriate locale detection strategy (Accept-Language header, user preferences, URL parameters).
- **R-API-006** SHOULD: For PDF generation, use streaming responses where possible to reduce memory footprint and improve time-to-first-byte.
- **R-API-007** SHOULD: Document URL parameter conventions in API specification (OpenAPI/Swagger) for external consumers.
- **R-API-008** SHOULD: Implement rate limiting and request validation middleware at the route level for public endpoints.
- **R-API-009** SHOULD: At least 90% of API routes SHOULD follow the established URL parameter conventions ($orgUrl, $teamUrl).
- **R-API-010** SHOULD: Lingui configuration SHOULD be present and integrated into route handlers for internationalized responses.
- **R-API-011** SHOULD: PDF generation utilities SHOULD be isolated in server-only packages with appropriate content-type handling.

### Verify

```bash
# Count authenticated route prefixes
grep -r '_authenticated+' apps/remix/app/routes/ | wc -l

# Count routes with loader or action functions
find apps/remix/app/routes -name '*.tsx' -exec grep -l 'loader\|action' {} \; | wc -l

# Verify Lingui configuration presence
grep -r 'lingui' . --include='*.config.*' --include='*.tsx' | head -5

# Find PDF generation utilities
find packages/lib/server-only -name '*pdf*.ts' -type f

# Verify settings route patterns
find apps/remix/app/routes -path '*settings*' -name '*.tsx' | head -10

# Check for _layout.tsx files with shared auth logic
find apps/remix/app/routes -name '_layout.tsx' | wc -l
```

**Accept when:**
- All authenticated routes use `_authenticated+` prefix and implement proper authorization checks
- At least 90% of API routes follow the established URL parameter conventions ($orgUrl, $teamUrl)
- Lingui configuration is present and integrated into route handlers for internationalized responses
- PDF generation utilities are isolated in server-only packages with appropriate content-type handling
- Settings endpoints follow the `o.$orgUrl.settings.{domain}` pattern for organizational settings
- Personal settings routes follow the `settings+/_dynamic_personal_routes+` pattern
- Shared authentication logic is implemented in `_layout.tsx` files
- TypeScript compilation succeeds without errors on all route handlers
