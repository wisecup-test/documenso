<rule_activation id="cffeea18-167b-4705-a4f7-7126b29e7bec" title="Adopt Server-Only Module Pattern for External API Integrations: Server Only Modules" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching `**/*`. Server-only modules must be strictly isolated from client-side code to prevent security vulnerabilities, API key exposure, and unauthorized access.
</rule_activation>

### Rules

- **R-SERVERONLY-001** SHOULD: Server-only modules SHOULD use framework-specific mechanisms (e.g., Remix loaders/actions, Next.js API routes) to expose functionality to client layers.

### Verify

```bash
# Check for client-side imports of server-only modules
grep -r "from.*server-only" apps/remix/app/routes --include="*.tsx" --include="*.ts" | grep -v "loader\|action" || echo "No client-side server-only imports detected"

# Find server-only modules exporting sensitive credentials
find packages/lib/server-only -type f -name "*.ts" -exec grep -l "export.*API_KEY\|SECRET\|PASSWORD" {} \; | wc -l

# Verify client bundle size and check for server-only code leakage
npm run build && du -sh dist/client/**/*.js | awk '{if($1 ~ /M/ && $1+0 > 5) print "Warning: Large client bundle detected: " $0}'
```

**Accept when:**
- All server-only modules reside under designated server-only directory structures (e.g., `packages/lib/server-only/`) and are not imported directly in client-side code
- Build process successfully prevents bundling of server-only code in client JavaScript outputs
- CI pipeline includes automated checks that fail on detection of server-only imports in client contexts
- All external API integrations, PDF operations, and audit log rendering are implemented as server-only modules
- ESLint rules and TypeScript path restrictions enforce server-only import boundaries
- No sensitive credentials (API keys, secrets, passwords) are exposed in client-accessible code

<enforcement>
Claude Code MUST NOT skip or defer verification. Build-time enforcement and CI checks are mandatory before deployment. Violations block deployment and require security team review.
</enforcement>
