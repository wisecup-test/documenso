<rule_activation id="49f59319-9f8e-44ca-8f01-515e3156aef9" title="Establish Type-Safe Public API Contracts for Library Modules: Server Only Modules" applies_to="**/*">
These rules are ALWAYS ACTIVE for all public API contract definitions and library module interfaces within the codebase, particularly for packages under /packages/, public API endpoints in /packages/api/, shared library modules in /packages/lib/, test constants in /packages/app-tests/, and OpenAPI specifications.
</rule_activation>

### Rules

- **R-API-001** MUST: Server-only modules MUST be isolated in dedicated directories (e.g., server-only/) to prevent client-side imports
- **R-API-002** MUST: All public API endpoints have corresponding TypeScript contract definitions with complete type coverage
- **R-API-003** MUST: Use TypeScript's 'export type' and 'export interface' to explicitly mark public contracts
- **R-API-004** MUST: Create contract.ts files in each package's public API directory to centralize type definitions
- **R-API-005** MUST: Maintain OpenAPI specifications for versioned API endpoints (v1, v2, etc.)
- **R-API-006** SHOULD: Leverage OpenAPI decorators or schema generators to keep OpenAPI specs synchronized with TypeScript types
- **R-API-007** SHOULD: Implement pre-commit hooks to validate that all public API functions have corresponding contract definitions
- **R-API-008** SHOULD: Establish contract review checklist and require JSDoc comments on all public interfaces
- **R-API-009** SHOULD: Use linting rules to enforce documentation on public contracts
- **R-API-010** SHOULD: Document migration path for existing APIs without contracts, prioritizing high-traffic endpoints first

### Verify

```bash
# Count contract definitions across packages
grep -r "export.*contract" packages/*/contract.ts | wc -l

# Find and verify OpenAPI and contract files
find packages/api -name 'openapi.ts' -o -name 'contract.ts' | xargs grep -l 'export'

# Verify TypeScript strict mode compilation
tsc --noEmit --strict && echo 'Type checking passed'

# Verify server-only module isolation
grep -r "server-only" packages/lib | grep -v node_modules
```

**Accept when:**
- All public API endpoints have corresponding TypeScript contract definitions with complete type coverage
- TypeScript strict mode compilation passes without type errors in contract files
- OpenAPI specifications exist for versioned API endpoints (v1, v2, etc.)
- Server-only modules are properly isolated and cannot be imported by client-side code
- Pre-commit hooks validate that all public API functions have corresponding contract definitions
- All public interfaces include JSDoc comments and documentation

<enforcement>
Claude Code MUST NOT skip or defer verification. TypeScript compiler in strict mode MUST pass during CI builds. ESLint rules MUST enforce contract file presence for public APIs. Code review MUST require contract definitions for new APIs. Automated tests MUST validate implementation matches contract specifications. Pull requests MUST be blocked if new public APIs lack contract definitions.
</enforcement>
