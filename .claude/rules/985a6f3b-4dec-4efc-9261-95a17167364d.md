<rule_activation id="985a6f3b-4dec-4efc-9261-95a17167364d" title="Enforce Input Validation for Data Access Operations: Data Access Operations" applies_to="packages/lib/server-only/**/*.ts">
These rules are ALWAYS ACTIVE for all data access operations involving user-supplied input parameters in server-only modules, including recipient updates, field modifications, and document operations.
</rule_activation>

### Rules

- **R-DAVO-001** MUST: Data access operations MUST sanitize string inputs to prevent injection attacks before constructing queries.

### Verify

```bash
# Count validation/sanitization references in server-only data access functions
grep -r 'validate\|sanitize\|check' packages/lib/server-only --include='*.ts' | wc -l

# Verify recipient and field operations contain validation before database operations
grep -r 'function.*update.*recipient\|function.*set.*field' packages/lib/server-only --include='*.ts' -A 5 | grep -c 'validate\|throw.*Error'

# Check input validation test coverage
npm run test -- --grep 'input validation' --reporter json | jq '.stats.passes'
```

**Accept when:**
- All data access functions in `packages/lib/server-only` modules contain input validation logic before database operations
- Validation tests exist for each data access function covering valid, invalid, and edge case inputs
- Code review checklist includes verification of input validation for all new data access functions
- No data access functions accepting user-supplied parameters lack sanitization or validation checks

<enforcement>
Claude Code MUST NOT skip or defer verification of input validation in data access operations. All new or modified data access functions must include validation tests and pass automated static analysis before merge.
</enforcement>
