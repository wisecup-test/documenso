<rule_activation id="32e736be-2214-48f0-aaa4-d478997f89c2" title="Standardize Logging Configuration for Environment-Aware Runtime Observability: Applications Provide Sensible" applies_to="**/*">
These rules are ALWAYS ACTIVE for all runtime components that require logging and observability capabilities.
</rule_activation>

### Rules

- **R-LOG-001** SHOULD: Applications SHOULD provide sensible default logging configurations that work without explicit environment variable settings.

### Verify

```bash
# Check for environment variable references for logging configuration
grep -r 'process\.env.*LOG' --include='*.ts' --include='*.tsx' --include='*.js' | wc -l

# Check for logger initialization or configuration code
grep -r 'logger\.configure\|createLogger\|winston\|pino' --include='*.ts' --include='*.tsx' | head -20

# Check for environment template files documenting logging configuration
find . -name '*.env.example' -o -name '*.env.template' | xargs grep -l 'LOG_LEVEL\|LOG_FORMAT' 2>/dev/null
```

**Accept when:**
- Environment variable references for logging configuration are found in multiple source files
- Logger initialization or configuration code is present in the codebase
- Environment template files document logging-related configuration variables

<enforcement>
Claude Code MUST verify logging configuration is environment-aware and MUST NOT skip verification of this rule.
</enforcement>
