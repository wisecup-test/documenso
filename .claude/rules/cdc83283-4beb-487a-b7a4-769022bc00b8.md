<rule_activation id="cdc83283-4beb-487a-b7a4-769022bc00b8" title="Adopt Remix File-Based Routing with URL Parameter Conventions for Authenticated Routes: Routes Handling Similar" applies_to="apps/remix/app/routes/**/*">
These rules are ALWAYS ACTIVE for all authenticated routes in the Remix application using file-based routing with URL parameter conventions for organization and team contexts.
</rule_activation>

### Rules

- **R-REMIX-001** SHOULD: Routes handling similar resources across different contexts (org vs team) SHOULD maintain parallel naming structures for consistency.
- **R-REMIX-002** MUST: All authenticated routes MUST be located within the `_authenticated+` directory segment.
- **R-REMIX-003** MUST: Organization routes MUST use the `o.$orgUrl` pattern consistently.
- **R-REMIX-004** MUST: Team routes MUST use the `t.$teamUrl+` pattern consistently.
- **R-REMIX-005** MUST: Index routes MUST use the `_index.tsx` suffix.
- **R-REMIX-006** MUST: Layout boundaries MUST use the `+` suffix appropriately.
- **R-REMIX-007** MUST: Route loaders MUST properly extract and validate URL parameters before accessing resources.
- **R-REMIX-008** SHOULD: Use Remix's `useParams()` hook to extract `$orgUrl` and `$teamUrl` parameters in loaders and components.
- **R-REMIX-009** SHOULD: Implement shared layout components at the `_authenticated+`, `o.$orgUrl`, and `t.$teamUrl+` boundaries to provide consistent navigation and context.
- **R-REMIX-010** SHOULD: Create utility functions for generating type-safe route URLs to avoid hardcoding paths.
- **R-REMIX-011** MAY: Legacy routes that predate this convention and require backward compatibility are excepted (EX-001).
- **R-REMIX-012** MAY: Experimental features under feature flags may use alternative routing patterns (EX-002).

### Verify

```bash
# Count authenticated routes with org/team patterns
find apps/remix/app/routes/_authenticated+ -name '*.tsx' | grep -E 'o\.\\$orgUrl|t\.\\$teamUrl' | wc -l

# Verify loaders extract params
grep -r 'export.*loader' apps/remix/app/routes/_authenticated+ | grep -c 'params'

# Count index routes
find apps/remix/app/routes/_authenticated+ -name '*_index.tsx' | wc -l
```

**Accept when:**
- All authenticated routes are located within the `_authenticated+` directory segment
- Organization routes use `o.$orgUrl` pattern and team routes use `t.$teamUrl+` pattern consistently
- Index routes use `_index.tsx` suffix and layout boundaries use `+` suffix appropriately
- Route loaders properly extract and validate URL parameters before accessing resources
- No protected resources are accessible outside the `_authenticated+` boundary

<enforcement>
Claude Code MUST verify route naming conventions and structure compliance before accepting changes to authenticated routes. Violations MUST be flagged and require explicit exception documentation with engineering lead approval.
</enforcement>
