<rule_activation id="df1f15e8-e8cd-45dd-8553-c1f77aa219cd" title="Establish Type-Safe Public API Contracts for Library Modules: Packages Use Constants" applies_to="**/*">
These rules are ALWAYS ACTIVE for all public API contract definitions and library module interfaces within the codebase, particularly for packages under /packages/ directory, public API endpoints in /packages/api/, shared library modules in /packages/lib/, and test constants in /packages/app-tests/.
</rule_activation>

### Rules

- **R-API-001** MAY: Packages MAY use constants files to define shared metadata and configuration for API contracts.
- **R-API-002** MUST: All public API endpoints have corresponding TypeScript contract definitions with complete type coverage.
- **R-API-003** MUST: Use TypeScript's 'export type' and 'export interface' to explicitly mark public contracts.
- **R-API-004** SHOULD: Create contract.ts files in each package's public API directory to centralize type definitions.
- **R-API-005** SHOULD: Leverage OpenAPI decorators or schema generators to keep OpenAPI specs synchronized with TypeScript types.
- **R-API-006** SHOULD: Implement pre-commit hooks to validate that all public API functions have corresponding contract definitions.
- **R-API-007** MUST: Server-only modules are properly isolated and cannot be imported by client-side code.
- **R-API-008** SHOULD: Establish contract review checklist and require JSDoc comments on all public interfaces.
- **R-API-009** SHOULD: Use linting rules to enforce documentation on public API contracts.
- **R-API-010** MUST: Enforce semantic versioning and maintain changelog for breaking changes to public contracts.

### Verify

```bash
# Count contract definitions across packages
grep -r "export.*contract" packages/*/contract.ts | wc -l

# Find and verify OpenAPI and contract files
find packages/api -name 'openapi.ts' -o -name 'contract.ts' | xargs grep -l 'export'

# Verify TypeScript strict mode compilation
tsc --noEmit --strict && echo 'Type checking passed'

# Verify server-only module isolation
grep -r "server-only" packages/lib | grep -v node_modules
```

**Accept when:**
- All public API endpoints have corresponding TypeScript contract definitions with complete type coverage
- TypeScript strict mode compilation passes without type errors in contract files
- OpenAPI specifications exist for versioned API endpoints (v1, v2, etc.)
- Server-only modules are properly isolated and cannot be imported by client-side code
- New public APIs include contract definitions verified by code review
- Pre-commit hooks validate contract presence for all public API functions

<enforcement>
Claude Code MUST verify TypeScript compilation in strict mode passes before accepting changes to public API contracts. Claude Code MUST NOT skip contract definition requirements for new public APIs. Violations block CI builds and pull requests until resolved.
</enforcement>
