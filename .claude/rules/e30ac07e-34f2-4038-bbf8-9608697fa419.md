<rule_activation id="e30ac07e-34f2-4038-bbf8-9608697fa419" title="Standardize Public API Contract Design with Type-Safe Route Handlers and Internationalization: Public Endpoints Not" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching the scope: Remix route handlers under apps/remix/app/routes/, server-side PDF generation utilities in packages/lib/server-only/, internationalization configuration files (lingui.config.ts), authentication-gated endpoints for organizations, teams, and personal settings, and document generation APIs.
</rule_activation>

### Rules

- **R-API-001** MUST_NOT: Public API endpoints MUST_NOT expose unauthenticated routes for organizational or personal data without explicit security review.
- **R-API-002** MUST: All authenticated routes use _authenticated+ prefix and implement proper authorization checks.
- **R-API-003** MUST: Use Remix's loader functions for GET requests and action functions for mutations, ensuring proper HTTP method semantics.
- **R-API-004** MUST: Implement shared authentication logic in _layout.tsx files to avoid duplication across route handlers.
- **R-API-005** SHOULD: At least 90% of API routes follow the established URL parameter conventions ($orgUrl, $teamUrl).
- **R-API-006** SHOULD: Configure Lingui with appropriate locale detection strategy (Accept-Language header, user preferences, URL parameters).
- **R-API-007** SHOULD: For PDF generation, use streaming responses where possible to reduce memory footprint and improve time-to-first-byte.
- **R-API-008** SHOULD: Document URL parameter conventions in API specification (OpenAPI/Swagger) for external consumers.
- **R-API-009** SHOULD: Implement rate limiting and request validation middleware at the route level for public endpoints.
- **R-API-010** MUST: Lingui configuration is present and integrated into route handlers for internationalized responses.
- **R-API-011** MUST: PDF generation utilities are isolated in server-only packages with appropriate content-type handling.

### Verify

```bash
# Count authenticated routes
grep -r '_authenticated+' apps/remix/app/routes/ | wc -l

# Find routes with loader or action functions
find apps/remix/app/routes -name '*.tsx' -exec grep -l 'loader\|action' {} \; | wc -l

# Verify Lingui configuration presence
grep -r 'lingui' . --include='*.config.*' --include='*.tsx' | head -5

# Find PDF generation utilities
find packages/lib/server-only -name '*pdf*.ts' -type f

# Verify TypeScript compilation
npm run type-check

# Run integration tests for authentication boundaries
npm run test:integration -- --testPathPattern=auth
```

**Accept when:**
- All authenticated routes use _authenticated+ prefix and implement proper authorization checks
- At least 90% of API routes follow the established URL parameter conventions ($orgUrl, $teamUrl)
- Lingui configuration is present and integrated into route handlers for internationalized responses
- PDF generation utilities are isolated in server-only packages with appropriate content-type handling
- TypeScript compilation succeeds without errors on all route handlers
- Integration tests validating authentication boundaries and authorization logic pass
- No unauthenticated routes access organizational or personal data without documented exceptions
- Code review checklist confirms internationalization support verification for new endpoints
