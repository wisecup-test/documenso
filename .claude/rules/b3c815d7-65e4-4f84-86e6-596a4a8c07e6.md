<rule_activation id="b3c815d7-65e4-4f84-86e6-596a4a8c07e6" title="Adopt Monorepo Package Structure with Scoped Modules: Shared Code Organized" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files in the monorepo to enforce consistent package structure organization and prevent architectural violations.
</rule_activation>

### Rules

- **R-MONO-001** MUST: All shared code MUST be organized under the packages/ directory with clear package scoping (e.g., packages/api, packages/lib, packages/app-tests).
- **R-MONO-002** MUST: Server-only code MUST be isolated in packages/lib/server-only to prevent accidental client-side imports.
- **R-MONO-003** MUST: API contract definitions and OpenAPI specifications MUST be located in packages/api with versioned subdirectories (e.g., packages/api/v1/).
- **R-MONO-004** MUST: No client-side code or components MAY import from packages/lib/server-only.
- **R-MONO-005** SHOULD: Use TypeScript path mapping in tsconfig.json to enable clean imports (e.g., '@packages/api/v1/contract' instead of relative paths).
- **R-MONO-006** SHOULD: Each package SHOULD include a README.md documenting its purpose and scope.
- **R-MONO-007** SHOULD: Establish naming conventions: packages/api for public contracts, packages/lib for shared utilities, packages/app-* for application-specific code.
- **R-MONO-008** MAY: Use monorepo management tools like Turborepo, Nx, or Lerna to manage builds, caching, and dependency orchestration.

### Verify

```bash
# Count TypeScript files in established package structure
find packages/ -type f -name '*.ts' | grep -E 'packages/(api|lib|app-tests)' | wc -l

# Verify no client-side imports of server-only code
grep -r 'packages/lib/server-only' --include='*.ts' --exclude-dir=node_modules | grep -v 'server-only' || echo 'No client-side imports of server-only code'

# Verify API contract structure exists
test -d packages/api/v1 && test -f packages/api/v1/openapi.ts && test -f packages/api/v1/contract.ts

# Validate no circular dependencies
madge --circular packages/ || echo 'Dependency graph validation required'
```

**Accept when:**
- All TypeScript files under packages/ directory follow the established structure with clear package boundaries (api, lib, app-tests)
- No imports of packages/lib/server-only code exist in client-side bundles or components
- API contracts and OpenAPI specifications are located in packages/api with versioned subdirectories
- Automated dependency graph validation passes without circular dependency errors
- Each package contains a README.md documenting its purpose and scope
- TypeScript path mappings are configured in tsconfig.json for clean imports
