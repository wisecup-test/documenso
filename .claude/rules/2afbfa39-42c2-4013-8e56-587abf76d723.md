<rule_activation id="2afbfa39-42c2-4013-8e56-587abf76d723" title="Adopt Server-Side Rendering with Loader Functions for Route-Level Data Fetching: Route Modules Export" applies_to="apps/remix/app/routes/**/*.{ts,tsx}">
These rules are ALWAYS ACTIVE for all Remix route modules. All route files that require server-side data for initial render MUST implement loader functions following the Remix framework pattern.
</rule_activation>

### Rules

- **R-SSR-001** MUST: All route modules MUST export a loader function when server-side data is required for initial render.
- **R-SSR-002** MUST: Route components MUST use Remix's useLoaderData() hook to access data returned from loader functions with full type safety.
- **R-SSR-003** MUST: Loader functions MUST implement error handling using throw new Response() for HTTP errors or throw redirect() for authentication redirects.
- **R-SSR-004** SHOULD: Leverage Remix's built-in caching headers in loader responses to enable CDN and browser caching where appropriate.
- **R-SSR-005** SHOULD: For routes requiring authentication, check session/token validity in the loader and redirect to login if unauthorized.
- **R-SSR-006** SHOULD: Use defer() for streaming server-side rendering when some data can be loaded asynchronously after initial render.
- **R-SSR-007** SHOULD: Structure loader functions to be testable independently from components, enabling unit tests for data fetching logic.
- **R-SSR-008** MUST NOT: Route components MUST NOT use useEffect with fetch or axios for initial data fetching.

### Verify

```bash
# Count exported loader functions in route files
grep -r "export.*loader" apps/remix/app/routes/ --include="*.tsx" --include="*.ts" | wc -l

# Detect client-side data fetching patterns in route components (should be 0)
grep -r "useEffect.*fetch\|useEffect.*axios" apps/remix/app/routes/ --include="*.tsx" | wc -l

# Verify TypeScript compilation passes
npm run type-check -- --noEmit
```

**Accept when:**
- All route files that require initial data have exported loader functions (grep count > 0 for routes with data needs)
- No route components use useEffect for initial data fetching (grep count = 0 for useEffect with fetch/axios in routes)
- TypeScript compilation passes without errors, confirming type safety between loaders and components
- Code review checklist verifies loader implementation for new routes
- ESLint custom rules detect and flag client-side data fetching patterns in route components
