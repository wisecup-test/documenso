<rule_activation id="b7468c90-59da-499e-b4f2-f287fbe498d9" title="Adopt E2E API Testing for Frontend Rendering Model Validation: Tests Include Performance" applies_to="**/*">
These rules are ALWAYS ACTIVE for all files matching API endpoints, test specifications, and CI/CD configurations that validate frontend rendering model contracts.
</rule_activation>

### Rules

- **R-E2E-001** MAY: Tests MAY include performance validation to ensure API response times meet frontend rendering requirements.
- **R-E2E-002** MUST: All REST API endpoints (v1, v2) that serve frontend applications require E2E test coverage.
- **R-E2E-003** MUST: All tRPC procedures that provide data for UI rendering require E2E test coverage.
- **R-E2E-004** MUST: API endpoints for document search, retrieval, and management require E2E test coverage.
- **R-E2E-005** MUST: API endpoints for template operations and field prefilling require E2E test coverage.
- **R-E2E-006** MUST: API endpoints for envelope distribution and validation require E2E test coverage.
- **R-E2E-007** MUST: API endpoints for embedding and presigning operations require E2E test coverage.
- **R-E2E-008** SHOULD: Organize E2E tests in directory structure matching API versioning: e2e/api/v1/, e2e/api/v2/, e2e/api/trpc/.
- **R-E2E-009** SHOULD: Use descriptive test names that clearly indicate the API endpoint and scenario being tested.
- **R-E2E-010** SHOULD: Create shared test utilities for common operations like authentication, test data setup, and response validation.
- **R-E2E-011** SHOULD: Implement test data factories to generate consistent, realistic test data for API requests.
- **R-E2E-012** SHOULD: Use API client libraries in tests to ensure tests exercise the same code paths as production frontend applications.
- **R-E2E-013** MUST: E2E test suite must run successfully in CI/CD pipeline with >90% pass rate.
- **R-E2E-014** MUST: Pull requests without corresponding E2E tests for API changes are blocked from merging.
- **R-E2E-015** MUST: CI/CD pipeline must fail if E2E test suite does not pass.

### Verify

```bash
# Count E2E API test specification files
find packages/app-tests/e2e/api -name '*.spec.ts' | wc -l | grep -E '^[8-9]|^[1-9][0-9]+$'

# Count total test cases across E2E API tests
grep -r 'describe\|test\|it' packages/app-tests/e2e/api --include='*.spec.ts' | wc -l

# List all E2E API test files
npm test -- --testPathPattern=e2e/api --listTests | wc -l
```

**Accept when:**
- At least 8 E2E API test specification files exist covering major API endpoints
- All API versions (v1, v2, tRPC) have corresponding test coverage for frontend-facing endpoints
- E2E test suite runs successfully in CI/CD pipeline with >90% pass rate
- Test coverage includes document operations, template management, envelope workflows, and search functionality
- Test file naming conventions follow pattern: `{feature}.spec.ts` (e.g., 'search-documents.spec.ts', 'template-field-prefill.spec.ts')
- Test organization matches API versioning structure: `e2e/api/v1/`, `e2e/api/v2/`, `e2e/api/trpc/`
