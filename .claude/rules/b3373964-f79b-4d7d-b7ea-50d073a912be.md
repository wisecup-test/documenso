<rule_activation id="b3373964-f79b-4d7d-b7ea-50d073a912be" title="Standardize Environment Variable Configuration Management Pattern: Environment Specific Configuration" applies_to="**/*">
These rules are ALWAYS ACTIVE for all runtime configuration and environment management implementations across server-side application code, API implementations, service integrations, PDF processing utilities, database connections, feature flags, and file system paths.
</rule_activation>

### Rules

- **R-ENV-001** MUST: Environment-specific configuration MUST be validated at application startup to fail fast if required values are missing or invalid.

### Verify

```bash
# Check for direct process.env access outside dedicated configuration modules
grep -r 'process\.env\.' --include='*.ts' --include='*.js' packages/ | grep -v 'config\|environment' | wc -l

# Verify centralized configuration modules exist
find packages/ -name 'config.ts' -o -name 'environment.ts' -o -name 'env.ts' | head -5

# Verify configuration imports are used in API and library code
grep -r 'import.*config.*from' --include='*.ts' packages/api packages/lib | wc -l
```

**Accept when:**
- Direct process.env access is minimal or zero outside of dedicated configuration modules
- At least one centralized configuration module exists that exports typed configuration objects
- Configuration imports are present in API and library code indicating usage of centralized configuration
- Configuration validation occurs at application startup with clear error messages for missing or invalid values
- .env.example files document all required and optional configuration variables

<enforcement>
Claude Code MUST verify all configuration management follows the centralized pattern. Code review checklist MUST include verification that new configuration uses centralized modules. ESLint rules MUST warn or error on direct process.env access outside configuration modules. CI pipeline MUST include verification commands to detect pattern violations. Pull requests with violations MUST be flagged and cannot merge to main branches without remediation or approved exceptions.
</enforcement>
