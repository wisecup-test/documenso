<rule_activation id="373b6b8e-91b2-48c7-9a9a-7d242d4393a5" title="Adopt Server-Side Rendering with Loader Functions for Route-Level Data Fetching: Route Components Not" applies_to="app/routes/**/*.{ts,tsx}">
These rules are ALWAYS ACTIVE for all Remix route modules in the application. All route components MUST implement server-side data loading patterns using loader functions as specified herein.
</rule_activation>

### Rules

- **R-SSR-001** MUST NOT: Route components MUST NOT fetch initial page data using client-side API calls when loader functions can be used.
- **R-SSR-002** MUST: All route files that require initial data MUST have exported loader functions.
- **R-SSR-003** MUST: Use Remix's useLoaderData() hook in route components to access data returned from loader functions with full type safety.
- **R-SSR-004** MUST: Implement error handling in loaders using throw new Response() for HTTP errors or throw redirect() for authentication redirects.
- **R-SSR-005** SHOULD: Leverage Remix's built-in caching headers in loader responses to enable CDN and browser caching where appropriate.
- **R-SSR-006** SHOULD: For routes requiring authentication, check session/token validity in the loader and redirect to login if unauthorized.
- **R-SSR-007** SHOULD: Use defer() for streaming server-side rendering when some data can be loaded asynchronously after initial render.
- **R-SSR-008** SHOULD: Structure loader functions to be testable independently from components, enabling unit tests for data fetching logic.

### Scope

In scope:
- All Remix route modules (files in app/routes directory)
- Public routes requiring initial data (_index.tsx)
- Authenticated routes with user-specific data
- Internal routes for server-side rendering (PDF generation, reports)
- SSO and authentication flow routes

Out of scope:
- Client-side only components that don't correspond to routes
- Shared utility components without route-level data requirements
- Real-time data subscriptions (WebSocket, SSE) for live updates
- Progressive enhancement features that load after initial render

### Exceptions

- **EXC-001**: Route requires real-time data updates that cannot be efficiently handled through revalidation
- **EXC-002**: Third-party widget or embed requires client-side initialization with dynamic data

### Verify

```bash
# Count exported loader functions in route files
grep -r "export.*loader" apps/remix/app/routes/ --include="*.tsx" --include="*.ts" | wc -l

# Detect client-side data fetching patterns in route components
grep -r "useEffect.*fetch\|useEffect.*axios" apps/remix/app/routes/ --include="*.tsx" | wc -l

# Verify TypeScript compilation
npm run type-check -- --noEmit
```

**Accept when:**
- All route files that require initial data have exported loader functions (grep count > 0 for routes with data needs)
- No route components use useEffect for initial data fetching (grep count = 0 for useEffect with fetch/axios in routes)
- TypeScript compilation passes without errors, confirming type safety between loaders and components
